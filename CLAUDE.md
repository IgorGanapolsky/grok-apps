# CTO Directive: grok-apps Standards

## Tech Stack
- React (Next.js) + TypeScript
- Vanilla CSS / CSS Modules
- Supabase (Backend)
- Playwright (Browser Automation)

## Engineering Standards
- **GSD (Get Shit Done):** Prioritize functional delivery.
- **Ralph Loop 247:** Continuous Execution -> Validation.
- **Surgical Edits:** Keep PRs focused.
- **Evidence-Based:** Every claim requires terminal output or CI proof.

## Operational Rules
1. Never claim completion without verification.
2. Log all lessons and mistakes to RAG.
3. Audit technical debt line-by-line every session.

## Security & Market Strategy (May 2026)
- **Zero Plaintext:** Never store API keys in unencrypted storage (use EncryptedSharedPreferences for Android).
- **Revenue Wedge:** Focus on "The Repeat Tax" ($400-$1,500/mo waste) as the primary enterprise pain point.
- **Direct Response:** Prioritize existing logged-in sessions (Safari, Comet) over incognito tabs to maintain distribution velocity.
